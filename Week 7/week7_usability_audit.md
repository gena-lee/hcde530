# UI3 Usability Audit

Analysis of 23 posts across Figma community forums (March – November 2025).  
Source: `week7_figma_feedback.csv` — UI3 migration thread & Editor UX critique thread.

---

## Problem 1 — Expert Workflow Disruption & Muscle Memory Loss
**Severity: Critical**

Long-time users report a complete collapse in working speed — from designing at near-automatic fluency to being functionally blocked — making UI3 unfit for professional daily use.

**Why Critical:** When a tool that power users rely on professionally causes them to feel paralyzed and actively spend personal time building workarounds, the core productivity contract is broken. The scale of this complaint (the highest-voted thread posts, multiple independent voices) confirms it is not fringe.

**Supporting posts:**

> "I've been using Figma for years and years — so much that I can create things almost at the speed of thought. With UI3 turned on I feel literally paralysed. I've tried turning UI3 on at least 50 times now but every time it's just as painful."
>
> — Darius Vitkunas, Post #11 (April 3, 2025)

> "UI3 is, without a doubt, the most unfriendly design possible. It has made my workflow painfully slow and incredibly unproductive."
>
> — OnlyRodas, Post #1 (April 2, 2025) · 111 helpful votes

> "UI3 is a bad update and your team undermining years of muscle memory for a bad update because of your egos is the worst."
>
> — Katelyn Ernesto, Post #13 (April 9, 2025)

## Problem 2 — Visual Accessibility & Input State Ambiguity
**Severity: High**

Multiple users independently flag thin/small type, invisible or ambiguous checkbox states, and an undifferentiated grey input panel — failures that violate basic accessibility standards and cause interaction errors.

**Why High:** Accessibility failures affect users with low vision directly, but the input-state ambiguity (inputs appearing selected when untouched, invisible checkboxes) causes errors for all users. Flagged by both casual and expert users, and by UX practitioners who noted the irony of a UX tool violating UX basics.

**Supporting posts:**

> "The new UI is not very accessible either — very small thin type. I really tried using the new UI and it is not as efficient or functional."
>
> — Artist and Machine, Post #7 (March 26, 2025)

> "[I need help] making checkboxes actually visible when I select them (I'm talking to you, 'Clip content')"
>
> — Darius Vitkunas, Post #11 (April 3, 2025)

> "I hate UI3 because of how visually noisy the right panel has become, with all grey backgrounds for inputs, extra fields, and no separation between sections."
>
> — Anastasia Zotov, Post #14 (April 9, 2025)

> "I do not appreciate how a lot of the values for the inputs look selected when I've not interacted with them."
>
> — Edward Saleeby, Post #10 (April 2, 2025)

## Problem 3 — Hidden & Buried Controls / Inconsistent Navigation
**Severity: High**

Key tools have been removed from persistent toolbars or buried in submenus, and UI element visibility depends on context rather than being consistently present — forcing re-discovery of known features and violating the principle of minimal surprise.

**Why High:** Hiding unavailable UI elements (rather than disabling them) removes the user's ability to learn the interface and understand why an action is unavailable. Burying frequently used tools like shape boolean operations in submenus adds friction to high-frequency professional tasks. Documented across both the UI3 migration thread and the broader editor UX critique thread.

**Supporting posts:**

> "Thanks for removing an entire navbar and moving the shape intersect/union and burying it in a submenu, but at least now we have an always-visible 'FigPal' button."
>
> — Radu Muntean, Post #15 (April 10, 2025)

> "So many functions are hidden below a small unlabeled icon. And when a function is unavailable, its UI element is hidden, not disabled. I'm using Figma for UX design, and it just feels so wrong to use a tool that does not handle UX basics correctly itself."
>
> — Martin Wermers-Pauly, Post #21 (September 17, 2025)

> "The position of things feels and looks drastically different depending on the complexity and options of components."
>
> — Edward Saleeby, Post #10 (April 2, 2025)
