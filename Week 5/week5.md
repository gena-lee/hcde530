## APIs and data acquisition (Last.fm)

**Base URL:** `https://ws.audioscrobbler.com/2.0/` — JSON responses with `method`, `api_key`, and `format=json` query parameters.

**Endpoints I relied on (by research question):**

| Question | Method | Role |
|----------|--------|------|
| RQ1 | [`geo.getTopArtists`](https://www.last.fm/api/show/geo.getTopArtists) | Top artists per **profile country** (US, Brazil, South Korea) |
| RQ2 | [`chart.getTopTags`](https://www.last.fm/api/show/chart.getTopTags) | Global tag chart; rank tags by **`reach`** |
| RQ3 | [`chart.getTopArtists`](https://www.last.fm/api/show/chart.getTopArtists) | Paginated **global** chart (100 artists); [`artist.getInfo`](https://www.last.fm/api/show/artist.getInfo), [`artist.getTopTags`](https://www.last.fm/api/show/artist.getTopTags) | Per-artist plays/listeners and **primary tag** |

**What I did:** Call **`lastfm_call`** from **`lastfm_api.py`** → parse nested JSON into pandas via **`pd.json_normalize`**, **`pd.concat`** for multi-page or multi-country pulls, **`pd.to_numeric(..., errors="coerce")`** where counts might be messy. On errors, Last.fm returns an **`error`** code in JSON—I surface that as a **`RuntimeError`** so cells fail visibly instead of silently.

**API keys:** Key stays in **environment or `.gitignore`d `.env`**—never pasted into notebooks that get pushed to GitHub.

**Claim:** I read Last.fm’s method docs, parameterized country/limit/page, and treated API responses as **analysis-ready tables** after normalization—same acquisition discipline as Week 4 (right endpoint, handle gaps), but with auth and pagination where needed.

## HCD reflection

Working from **live APIs** means designs (and analyses) inherit **bias**: Last.fm **country** is **profile location**, not census geography; **tags** are community vocabulary, not a record-label genre tree; **`reach`** and **plays** are **platform metrics** that move over time. The honest loop is the same as Week 4: choose endpoints that match the question, document **missing or coarse fields**, and separate **full** pulls from **filtered** tables when transparency matters—in MP1, deduplicating chart rows and skipping artists without listeners/tags mirrors keeping both “all” and “cleaned” exports.

For stakeholders, I would pair **every chart** with **what was excluded** (e.g. tags below the artist-count threshold) so conclusions read as **bounded claims**, not universal truths.
